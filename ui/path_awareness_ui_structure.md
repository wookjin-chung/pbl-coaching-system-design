# Path Awareness UI Structure  
## 경로 인식을 드러내는 UI 구조 설계

---

## 이 문서의 목적

이 문서는 **Path Awareness Layer를 UI 차원에서 어떻게 구현할 것인지**를 정의합니다.

여기서 말하는 UI 구조는:
- 화면 레이아웃 스펙이 아니라
- 컴포넌트의 위치 목록이 아니라
- **사용자가 스스로 자신의 경로를 인식하게 만드는 배치 원칙**입니다.

이 문서는 “무엇을 보여줄 것인가”보다  
👉 **“언제, 얼마나, 어떤 방식으로 보여주지 않을 것인가”**를 더 중요하게 다룹니다.

---

## 기본 전제

### 1️⃣ 경로 인식은 ‘설명’이 아니라 ‘느낌’이다

사용자는:
- “당신은 지금 이 단계에 있습니다”라는 설명보다
- “아, 내가 여기까지 왔구나”라는 **자기 인식**을 통해
경로를 인식해야 합니다.

UI는 이를 위해:
- 직접적인 단계 선언을 피하고
- 암시적 구조와 배치로만 경로를 드러냅니다.

---

### 2️⃣ 모든 경로 요소를 동시에 보여주지 않는다

Path Awareness UI는  
과거·현재·미래를 **동시에 전시하지 않습니다**.

이는:
- 정보 과잉을 막고
- AI나 시스템이 판단을 대신하는 인상을 줄이며
- 사용자의 내적 판단을 흐릴 수 있기 때문입니다.

---

## 경로 인식 UI의 3대 구성 요소

Path Awareness UI는  
항상 다음 세 요소 중 **하나만 활성화**합니다.

1. 과거 카드 (Past Card)  
2. 현재 상태 (Present State)  
3. 미래 씨앗 (Future Seed)

이 셋은 **동시에 중심에 오지 않습니다.**

---

## 1️⃣ 과거 카드 (Past Card)

### 역할

과거 카드는:
- “지금의 혼란이 무의미하지 않다”는 감각을 제공하고
- 사용자가 **출발 지점과 누적 경로를 재연결**하도록 돕습니다.

---

### 노출 조건

과거 카드는 다음 상황에서만 등장합니다.

- 장기 중단 후 재방문 시
- 사용자가 명시적으로 “되돌아보기”를 선택했을 때
- 시스템이 **자기 위치 상실 신호**를 감지했을 때

---

### 표현 원칙

- 요약은 **문장 1~2개 이하**
- 평가/분석 문구 금지
- “이전에 이런 판단을 했다” 수준의 **사실 중심 서술**

❌ “이 판단은 좋았다 / 나빴다”  
⭕ “이 시점에 이런 선택을 기록했다”

---

## 2️⃣ 현재 상태 (Present State)

### 역할

현재 상태는:
- 사용자가 “지금 무엇을 하고 있는지”를  
  **자기 언어로 느끼게** 만드는 장치입니다.

현재 상태는  
단계 이름이 아니라 **행동 맥락**으로 표현됩니다.

---

### 노출 조건

현재 상태는:
- 사용자가 기록을 작성 중일 때
- 수정·정리·되돌림을 하고 있을 때
- 별도의 개입 없이 **항상 기본적으로 유지**됩니다.

---

### 표현 원칙

- 단계 명칭 사용 금지  
  (탐색/몰입/자각/정리)
- “지금 당신은 ___ 중입니다” 같은 선언 금지

대신:
- 입력 필드의 형태
- 버튼의 개수
- 주변 정보의 밀도

로 **현재 맥락을 암시**합니다.

---

## 3️⃣ 미래 씨앗 (Future Seed)

### 역할

미래 씨앗은:
- 다음 행동을 지시하지 않고
- 계획을 요구하지 않으며
- **열린 방향의 존재만을 암시**합니다.

이는 사용자가  
결론을 서두르지 않도록 돕는 장치입니다.

---

### 노출 조건

미래 씨앗은 다음 경우에만 나타납니다.

- 사용자가 하나의 기록을 마쳤을 때
- AI 응답 이후 짧은 여백 구간
- 장기 정체 상태가 감지될 때

---

### 표현 원칙

- 질문형 문장만 허용
- 행동 지시 금지
- 선택지는 **1~2개 이하**

❌ “다음으로 이것을 하세요”  
⭕ “이 방향을 나중에 다시 살펴볼 수도 있습니다”

---

## 세 요소 간 전환 규칙

Path Awareness UI는  
다음 전환 규칙을 따릅니다.

- 과거 카드가 중심일 때 → 미래 씨앗 비활성
- 미래 씨앗이 중심일 때 → 과거 카드 숨김
- 현재 상태는 항상 배경에 존재하되  
  **중심으로 부각되지 않을 수 있음**

이 규칙은  
**사용자의 판단 공간을 확보**하기 위한 장치입니다.

---

## UI 상태 머신과의 연결

이 구조는  
`ui_state_machine.md`의 상태들과 다음처럼 연결됩니다.

- `Resumable` → 과거 카드 중심
- `Writing / Editing` → 현재 상태 중심
- `Guided 이후` → 미래 씨앗 잠깐 노출

UI는 이 전환을  
애니메이션이나 강조로 과장하지 않습니다.

---

## AI 응답과의 결합 원칙

AI 응답은:
- 활성화된 경로 요소에만 반응합니다.

예:
- 과거 카드 활성 → 요약/재연결 질문
- 현재 상태 중심 → 입력을 방해하지 않는 최소 질문
- 미래 씨앗 활성 → 열린 질문 1개

AI는  
다른 요소를 **동시에 끌어오지 않습니다.**

---

## 실패·중단·정체를 다루는 UI 태도

Path Awareness UI는:

- 중단을 **문제 삼지 않으며**
- 정체를 **교정하려 들지 않고**
- 반복을 **비효율로 취급하지 않습니다.**

UI는 단지:
👉 **“다시 돌아올 수 있는 자리”를 남깁니다.**

---

## 설계 원칙 요약

- 경로는 요약하지 않는다
- 위치는 선언하지 않는다
- 미래는 지시하지 않는다
- 과거는 평가하지 않는다
- 현재는 방해하지 않는다

---

## 한 문장 요약

> **Path Awareness UI 구조는  
> 사용자를 앞으로 끌고 가는 인터페이스가 아니라,  
> 사용자가 스스로 방향을 되찾게 하는 공간 배치다.**
